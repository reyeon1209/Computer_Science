### 1. 데이터 통신 시스템의 구성요소
- 송신자(source)와 수신자(destination)
- 전송매체(media)
- 메시지(message)



## **THEMA 37 데이터 전송 기술**

### 1. 전송 속도 단위
- BPS (Bit Per Second): 디지털 단위, 1초 동안에 전송된 비트수
- Baud(보) : 아날로그 단위, 1초 동안에 전송된 주기(cycle)의 수
- BPS = Baud * cycle 당 전송 비트 수
- 2위상 = 1비트, 4위상 = 2비트, 8위상 = 3비트, 16위상 = 4비트

### 2. 데이터 전송 방식
#### (1) 단방향과 양방향 전송 방식
- 단방향 통신 (Simplex): 한 방향으로만 전송 가능
- 반이중 통신 (Half Duplex): 양방향 통신 가능 (동시에는 불가능), 데이터 흐름을 바꾸는 전송 반전 시간 필요
- 전이중 통신 (Full Duplex): 두 개의 통신 채널을 이용하여 동시에 데이터 송수신 가능

#### (2) 직렬 전송과 병렬 전송
- 직렬 전송: 하나의 전송로, 순차적 송신, 느림, 원거리 전송
- 병렬 전송: 각각의 전송로, 빠름, 근거리 전송

#### (3) 비동기식 전송과 동기식 전송
- 비동기식 전송 (start-stop 전송): 문자 단위, 유휴 시간(idle time) 발생, 시작비트 & 정지비트 & 패리티비트
- 동기식 전송: 블록 단위, 문자지향 - BSC(이중패리티), 비트지향 - CRC(순환중복검사)
- 비동기식 전송: 셀 단위, 초고속

### 3. 정보 전송의 변조 방식
#### (1) PCM (펄스 부호 변조)
- 표본화(Sampling): 연속적인 아날로그 정보에서 일정 시간마다 신호값을 추출
- 양자화(Quantization): 신호값에서 대표 10진수를 얻음
- 부호화(Encoding): 양자화 과정에서 얻은 결과값을 2진수로 변환

#### (2) 디지털 변조
- 진폭편이 변조(ASK, Amplitude)
- 주파수편이 변조(FSK, Frequency)
- 위상편이 변조(PSK, Phase)
- 진폭위상편이 변조(QAM)



## **THEMA 38 다중화(Multiplexing)**

### 1. 다중화 방식
#### (1) FDMA 주파수 분할 
- 아날로그 전송
- 공간 분할
- 전송 매체의 대역폭이 넓을 때
- 보호대역 필요

#### (2) TDMA 시분할
- 동기식 시분할 (STDMA): 고정된 타임슬롯
- 비동기식 시분할 (ATDMA): 유동적으로 타임슬롯 할당, 링크 효율 높임, 송신 터미널을 나타내는 주소 필요

#### (3) CDMA 코드 분할
- 무선 디지털
- 주파수 대역 확산 기법 (Spread Spectrum)
- 시간과 주파수 공유

#### (4) OFDMA 직교 주파수 분할
- 무선 디지털
- 직교성을 갖는 여러 주파수에 데이터를 나누어 전송



## **THEMA 39 Protocol (프로토콜)**

### 1. 프로토콜
- 통신 프로토콜이란 통신 개체 사이의 보다 효율적이고 정확한 데이터의 송수신을 위해 필요한 일연의 절차나 규범의 집합을 말한다.

### 2. 프로토콜의 구성 요소
- 구문(Syntax)
- 의미(Semantic)
- 순서(Timing)

### 3. 프로토콜의 주요 기능
- 단편화(Fragmentation)
- 재합성(Reassembly)
- 캡슐화(Encapsulation)
- 연결제어(Connection Control)
- 흐름제어(Flow Control)
- 에러제어(Error Control)
- 순서제어(Sequencing Control)
- 동기화(Synchronization)
- 주소설정(Addressing)



## **THEMA 40 OSI 참조 모델**

### 1. OSI 7계층
#### (1) 응용
- 사용자에게 제공하는 서비스
- PDU: 메시지(message)

#### (2) 표현
- 코드 변환, 암호화, 압축

#### (3) 세션
- 프로세스 간 연결/전달/해제, 대화관리, 복구를 위한 동기점 관리, 전이중 또는 반이중 방식 결정

#### (4) 전송
- 신뢰성 있는 종단 간 전송 보장, **연결제어**/오류제어/흐름제어/순서제어, 포트번호
- PDU: 세그먼트(segment)

#### (5) 네트워크
- IP주소(논리주소), 최적의 경로 설정
- PDU: 패킷(packet)

#### IGP (내부 게이트웨이 프로토콜)
- RIP: 거리 벡터 알고리즘 사용, 주기적 변경 신호 전송, 경유할 가능성이 있는 라우터를 홉(hop)수로 수치화하여 사용
- OSPF: 링크 상태 알고리즘(다익스트라 알고리즘) 사용, 변경 시에만 변경 신호 전송, 멀티캐스팅 지원

#### EGP (외부 게이트웨이 프로토콜)
- BGP: 경계 게이트웨이 프로토콜

#### (6) 데이터링크
- MAC주소(물리주소), 이웃노드 간 전송 보장, 오류제어/흐름제어/순서제어
- PDU: 프레임(frame)

#### LLC(논리 링크 제어)
- IEEE 802.2
- BSC(Binary Synchronous Control), HDLC(High-level Data Link Control)

#### MAC(매체 접근 제어)
- CSMA/CD(802.3), Token Bus(802.4), Token Ring(802.5)
- CSMA/CA(802.11b), Bluetooth(802.15.1)
- Wibro(802.16)

#### (7) 물리
- 전송 매체 (무선, 유선)
- PDU: 비트(bits)



## **THEMA 41 TCP/IP 인터넷 4계층 구조**

### 1. 응용 (7, 6, 5)
- HTTP: 웹(WWW), TCP 80
- SMTP: 메일송신, TCP 25
- POP3: 메일수신, TCP 110
- IMAP: 메일수신, TCP 143

- SNMP: 망 관리, UDP 161
- DNS: 도메인-IP주소, UDP 53
- RTP: 실시간 멀티미디어 전송
- TFTP: 간이 파일 전송, UDP 69

### 2. 전송 (4)
- TCP: 프로세스 간 정보 전달을 위해 포트 번호를 이용
- UDP
- SCTP

### 3. 인터넷 (3)
- ICMP: 인터넷 제어 메시지 생성
- IGMP: 그룹 관리 (멀티캐스트)

### 4. 네트워크 접속 (2, 1)

## **THEMA 43 IP**

### 1. IP 특징
- 흐름제어 불가
- 비연결성 데이터그램 방식
- 비신뢰성

### 2. IPv4주소와 IPv6주소 특징 비교

| |IPv4|IPv6|
|---|---|---|
|주소길이|32비트|128비트|
|주소표기방법|8비트*4개필드, 10진수 표기|16비트*8개필드, 16진수 표기, :로 구분|
|주소할당|클래스 단위의 비순차적 할당|네트워크 규모, 단말 수에 따른 순차적 할당|
|캐스트방식|유니, 멀티, **브로드**|유니, 멀티, **애니**|
|보안|별도 구현|IPSec 내장|
|헤더길이|20~60바이트 가변|40바이트 고정|

### 3. 사설 IP 주소
- A클래스: 10.0.0.0 ~ 10.255.255.255
- B클래스: 172.16.0.0 ~ 172.31.255.255
- C클래스: 192.168.0.0 ~ 192.168.255.255

### 4. IPv4 헤더
- 식별자
- 단편화 오프셋: 8바이트 단위
- TTL
- 헤더 체크섬

### 5. IPv6 헤더
- 우선순위
- 흐름 레이블
- 홉 제한 (Hop Limit)

### 6. IPv4에서 IPv6로의 천이 전략
- 이중 스택: 동시에 사용
- 터널링: IPv4를 사용하는 네트워크 영역을 통과해야 할 때
- 헤더 변환: 수신측에서 IPv6를 이해하지 못하는 경우



## **THEMA 44 ICMP 메시지**

### 1. ICMP(Internet Control Message Protocol) 질의 메시지
- Echo Request & Reply
- Timestamp Request & Reply
- Address Mask Request & Reply
- Router Solicitation Request & Reply

### 2. ICMP 오류 메시지
- ICMP Destination Unreachable 메시지: 라우터가 특정 노드의 패킷을 목적지에 보내지 못할 경우
- ICMP Redirect 메시지: 라우터가 송신측 노드에 적합하지 않은 경로로 설정되어 있는 경우, 해당 노드에 대한 최적화 경로를 다시 지정
- ICMP Time Exceeded 메시지: 패킷의 TTL이 도중에 0이 되어 폐기되는 경우
