## **THEMA 37 데이터 전송 기술**

### 1. 전송 속도 단위
- BPS (Bit Per Second): 디지털 단위, 1초 동안에 전송된 비트수
- Baud(보) : 아날로그 단위, 1초 동안에 전송된 주기(cycle)의 수
- BPS = Baud * cycle 당 전송 비트 수
- 2위상 = 1비트, 4위상 = 2비트, 8위상 = 3비트, 16위상 = 4비트

### 2. 데이터 전송 방식
#### (1) 단방향과 양방향 전송 방식
- 단방향 통신 (Simplex): 한 방향으로만 전송 가능
- 반이중 통신 (Half Duplex): 양방향 통신 가능 (동시에는 불가능), 데이터 흐름을 바꾸는 전송 반전 시간 필요
- 전이중 통신 (Full Duplex): 두 개의 통신 채널을 이용하여 동시에 데이터 송수신 가능

#### (2) 직렬 전송과 병렬 전송
- 직렬 전송: 하나의 전송로, 순차적 송신, 느림, 원거리 전송
- 병렬 전송: 각각의 전송로, 빠름, 근거리 전송

#### (3) 비동기식 전송과 동기식 전송
- 비동기식 전송 (start-stop 전송): 문자 단위, 유휴 시간(idle time) 발생, 시작비트 & 정지비트 & 패리티비트
- 동기식 전송: 블록 단위, 문자지향 - BSC(이중패리티), 비트지향 - CRC(순환중복검사)
- 비동기식 전송: 셀 단위, 초고속

### 3. 정보 전송의 변조 방식
#### (1) PCM (펄스 부호 변조)
- 표본화(Sampling): 연속적인 아날로그 정보에서 일정 시간마다 신호값을 추출
- 양자화(Quantization): 신호값에서 대표 10진수를 얻음
- 부호화(Encoding): 양자화 과정에서 얻은 결과값을 2진수로 변환

#### (2) 디지털 변조
- 진폭편이 변조(ASK, Amplitude)
- 주파수편이 변조(FSK, Frequency)
- 위상편이 변조(PSK, Phase)
- 진폭위상편이 변조(QAM)



## **THEMA 38 다중화(Multiplexing)**

### 1. 다중화 방식
#### (1) FDMA 주파수 분할 
- 아날로그 전송
- 공간 분할
- 전송 매체의 대역폭이 넓을 때
- 보호대역 필요

#### (2) TDMA 시분할
- 동기식 시분할 (STDMA): 고정된 타임슬롯
- 비동기식 시분할 (ATDMA): 유동적으로 타임슬롯 할당, 링크 효율 높임, 송신 터미널을 나타내는 주소 필요

#### (3) CDMA 코드 분할
- 무선 디지털
- 주파수 대역 확산 기법 (Spread Spectrum)
- 시간과 주파수 공유

#### (4) OFDMA 직교 주파수 분할
- 무선 디지털
- 직교성을 갖는 여러 주파수에 데이터를 나누어 전송



## **THEMA 39 Protocol (프로토콜)**

### 2. 프로토콜의 구성 요소
- 구문(Syntax)

- 단편화(Fragmentation)
- 재합성(Reassembly)
- 캡슐화(Encapsulation)
- 연결제어(Connection Control)
- 흐름제어(Flow Control)
- 에러제어(Error Control)
- 순서제어(Sequencing Control)
- 동기화(Synchronization)
- 주소설정(Addressing)

#### EGP (외부 게이트웨이 프로토콜)

#### LLC(논리 링크 제어)

#### MAC(매체 접근 제어)

## **THEMA 43 IP**

### 1. IP 특징
- 흐름제어 불가
- 비연결성 데이터그램 방식
- 비신뢰성

### 2. IPv4주소와 IPv6주소 특징 비교

| |IPv4|IPv6|
|---|---|---|
|주소길이|32비트|128비트|
|주소표기방법|8비트*4개필드, 10진수 표기|16비트*8개필드, 16진수 표기, :로 구분|
|주소할당|클래스 단위의 비순차적 할당|네트워크 규모, 단말 수에 따른 순차적 할당|
|캐스트방식|유니, 멀티, **브로드**|유니, 멀티, **애니**|
|보안|별도 구현|IPSec 내장|
|헤더길이|20~60바이트 가변|40바이트 고정|

### 3. 사설 IP 주소
- A클래스: 10.0.0.0 ~ 10.255.255.255

### 5. IPv6 헤더
- 우선순위

### 6. IPv4에서 IPv6로의 천이 전략

## **THEMA 44 ICMP 메시지**

### 1. ICMP(Internet Control Message Protocol) 질의 메시지
- Echo Request & Reply
- Timestamp Request & Reply
- Address Mask Request & Reply
- Router Solicitation Request & Reply

### 2. ICMP 오류 메시지
- ICMP Destination Unreachable 메시지: 라우터가 특정 노드의 패킷을 목적지에 보내지 못할 경우
- ICMP Redirect 메시지: 라우터가 송신측 노드에 적합하지 않은 경로로 설정되어 있는 경우, 해당 노드에 대한 최적화 경로를 다시 지정
- ICMP Time Exceeded 메시지: 패킷의 TTL이 도중에 0이 되어 폐기되는 경우



## **THEMA 45 제어 방식**

### 1. 전송 제어 / 흐름 제어
#### (1) 전송 제어
#### (2) 흐름 제어
- Stop & Wait
- 슬라이딩 윈도

### 2. 에러 제어
#### (1) 패리티 비트 검사
- 비동기 전송

#### (2) 블록 합 검사 (BSC)
- 문자지향 동기

#### (3) 순환 중복 검사 (CRC)
- 비트지향 동기
- 모듈로-2 연산 사용

#### (4) 체크섬 (Checksum)
- TCP, UDP, IPv4 헤더
- 데이터를 n bit로 잘라서 합한 후 1의 보수
- 원본 데이터와 체크섬을 함께 전송

#### (5) 해밍 코드 검사
- 1비트의 에러 검출 후 정정

### 3. 에러 정정 방식
#### (1) 검출 후 재전송(ARQ) 방식
- Stop & Wait: 반이중, ACK / NAK를 기다린다.
- Go-Back N: 전이중, 오류 프레임부터 나머지 프레임까지 다시 송신
- Selective(선택적): 전이중, 에러가 발생한 프레임만 다시 송신
- Adaptive(적응적): 에러 발생률이 낮으면 긴 프레임, 높으면 짧은 프레임 전송

#### (2) 전진 에러 수정(FEC: Forward Error Correction) 방식
- 해밍코드를 사용해 수신측에서 에러 정정
- 역채널 필요 X

### 2. IT 융합기술
#### (1) IP-TV
- 인터넷을 이용하여 방송 및 기타 콘텐츠를 TV로 제공하는 서비스 방식

#### (2) DMB
- 이동 멀티미디어 방송

#### (3) Wibro
- 언제 어디서나 이동 중에도 높은 전송속도로 무선인터넷 접속이 가능한 서비스

#### (4) RFID
- 무선 주파수를 이용한 비접촉 인식장치

#### (5) USN (Ubiquitous Sensor Network)
- 무선 식별 시스템
- 대상 물체에 부착된 지능칩과 다양한 센서들을 통해 언제 어디서나 원하는 정보를 수집하고 통신

#### (6) ITS (Intelligent Transport System)
- 기존 교통체계의 구성요소에 첨단 전자, 정보, 통신, 제어기술들을 적용

#### (7) 텔레매틱스
- 자동차 내 단말기를 통해 자동차와 운전자에게 다양한 정보 서비스 제공

#### (8) NFC (Near Field Communication)
- 근거리 무선 통신

#### (9) Bluetooth

#### (1) Beacon(비콘)
- 근거리에 있는 스마트 기기를 자동으로 인식하여 필요한 데이터 전송
