### 13. 컴퓨터 기억용량/저장용량

### 121. 부울의 대수

### 133. 부울식 간략화
- F = A'B'C + A'BC' + AB'C' + ABC = A'(B'C+BC') + A(B'C'+BC) = A'(B⊕C) + A(B⊕C)' = A'Z + AZ' = A⊕Z = A⊕B⊕C  
- AB + A'C + ABD' + A'CD' + BCD' = AB(1+D') + A'C(1+D') + BCD' = AB + A'C + BCD' = AB + A'C + **(A+A')** BCD' = AB + A'C + ABCD' + A'BCD' = AB(1+CD') + A'C(1+BD) = AB + A'C


### 148. 전가산기 논리회로
- 캐리를 포함하여 연산처리하기 위해 설계
- 2개의 반가산기와 OR 게이트로 구성
- S = (A⊕B)⊕Ci
- V = 1이면 오버플로우 발생


### 152. 회로
- 조합논리회로: 3개의 입력 중에서 적어도 2개의 입력이 1이면 출력이 1이 되는 회로 ex) 멀티플렉서, 전가산기, 디코더
- 순차논리회로: 플립플롭을 포함하여 이전 값을 기억하여 입력값과 함께 다음 출력값을 결정하는 논리회로 ex) 카운터, 레지스터


### 178. 명령어
- 컴퓨터 시스템의 명령어 형식: OPCODE + OPERAND = 32비트
- OPERAND 필드가 256M 워드 크기의 메모리 주소를 나타냄
- 256M 워드 = 2^28 워드 → OPERAND 길이: 28비트
- OPCODE 길이: 32 - 28 = 4비트 → OPCODE 명령어: 2^4 = 16개


### 183. 서브루틴 호출 (Subroutine Call)
- 호출되는 루틴을 사용자 프로그램이 선택
- 프로그램 명령어 실행에 의해서만 발생

#### (2) 비동기/외부 인터럽트

### 194. 컴퓨터 버스
- 3-state 버퍼: 데이터를 송신하고 있지 않은 장치의 출력이 버스에 연결된 다른 장치와 간섭하지 않도록 분리 가능


### 202. 250. SRAM / DRAM
- SRAM: 캐시메모리에 사용
- DRAM: 주기억장치에 사용, 캐퍼시터 사용 → 주기적 refresh 필요


### 205. 주기억장치의 용량
- 주기억장치의 용량 = 주소 개수(워드 개수) * 한 워드의 크기
- 주소 개수 = 2^(MAR의 길이) = 2^(PC의 길이)
- 한 워드의 크기 = MBR의 길이


### 233. 2-way 집합 연관 사상
- 주기억장치 주소 = 태그 8비트 + 세트 9비트 + 오프셋 7비트
- 캐시 기억장치의 라인 수 = 2^(세트 필드 길이) * 2(way) = 2^9 * 2 = 2^10


### 235. 캐시 일관성(Cache Coherence) 문제
- 소프트웨어 해결법
- 하드웨어 해결법: 디렉터리 기반 프로토콜, 스누핑 프로토콜, MESI(Modified, Exclusive, Share, Invalid) 프로토콜


### 236. 캐시 기억장치
- 명령어 캐시 기억장치 & 데이터 캐시 기억장치로 분리 가능
- 쓰기 버퍼: 즉시 쓰기 캐시 기억장치에서 쓰기 동작이 오래 걸리는 문제 개선


###  238. 메모리 인터리빙 (Memory Interleaving)
- 기억장치 모듈을 여러 개 가진 기억장치로 하나의 사이클 타임에 복수개의 단어를 시분할로 판독 → 기억장소 접근 효율 증가


### 243. 가상 메모리
- 페이지 공유를 통해 두 개 이상의 프로세스들이 메모리 공유
- TLB(Translation-Lookaside Buffer): 가상메모리 주소 전환을 효율적으로


### 248. NAND flash 메모리
- 하드 디스크와 유사하게 동작
- read와 write가 page 단위로 수행, page 크기: 섹터 크기의 배수
- erase가 block 단위로 수행


### 252. SSD
- HDD보다 가벼움
- 접근 속도 빠름
- 전력 소모 적음


### 253. RAID (Redundant Array of Inexpensive Disks)
- 데이터를 스트라이핑하여 프로세서와 디스크 드라이브 사이의 속도 차이를 개선
- RAID0: 스트라이핑
- RAID1: 미러링 중복 저장
- RAID2: 비트단위, 해밍코드
- RAID3: 비트단위, 패리티 정보
- RAID4: 블록단위, 패리티디스크 → 병목 현상
- RAID5: 블록단위, 패리티 블록 분산
- RAID6: 블록단위, 이중 패리티


### 265. I/O 장치가 시스템 버스에 직접 접속되지 못하는 이유
- 종류에 따라 제어방법이 서로 다른 I/O 장치들의 제어 회로들을 CPU 내부에 모두 포함시키는 것이 어려워 CPU가 그들을 직접 제어할 수 없기 때문
- I/O 장치들과 CPU가 사용하는 데이터 형식의 길이가 서로 다른 경우가 많기 때문


### 271. 네트워크를 통한 데이터 전송
- 스트로브 (Strobe) 제어 방법
- 핸드쉐이킹 (Handshaking) 방식


### 273. 버퍼링 / 스풀링
|버퍼링(Buffering)|스풀링(Spooling)|
|---|---|
|입출력 장치 속도 보완|입출력 장치 속도 보완|
|**한** 작업의 입출력 & 계산 동시 수행|**여러** 작업의 입출력 & 계산 동시 수행|
|주기억장치 사용|디스크 사용|
|스택, 큐 방식|큐 방식|


### 278. DMA 데이터 전송 기법
- 대량의 데이터 전송의 경우 인터럽트 방식보다 DMA 방식이 더 효율적


### 297. CPU 관련 기술
- VLIW(Very Long Instruction Word): 하나의 명령어 형식에 다수의 연산코드를 포함, 여러 개의 기능 유닛에서 동시에 연산 실행


### 313. 314. 컴퓨팅 시스템
- 불균일 기억장치 액세스(NUMA) 방식: 멀티프로세서 시스템에서 사용하는 컴퓨터 메모리 설계 방법
- 그리드 컴퓨팅 시스템: PC, 워크스테이션, 서버 등과 같은 다양한 컴퓨터들의 네트워크


### 317. 컴퓨터 인터페이스
- SATA(Serial Advanced Technology Attachment): 고속 전송이 가능한 직렬버스 형태 하드디스크 연결용 접속 규격


### 321. 벡터 방식 / 비트맵 방식
- 비트맵 방식: 이미지 크기와 저장 용량 간 관계


### 322. 324. 이미지 파일 형식
- WMF: 벡터방식을 지원하기 위한 공통적 형식
- PSD: 포토샵의 기본적인 파일 형식
- GIF: RLE 방식 응용한 압축 방법, 256색만 사용 가능


### 328. 이미지 저장 용량
- 256가지 색상의 1024(픽셀)*1024(픽셀) 사진 저장 용량?
- 256 = 2^8 → 1화소가 8비트
- 1024(화소) * 1024(화소) * 8비트(1화소 당 비트수) = 1MB


### 331. 멀티미디어 데이터 압축
- MPEG-1: CD-ROM과 같은 기록 매체에 VHS 테이프 수준의 동영상과 음향을 최대 1.5Mbps로 전송 가능하도록 압축하는 규약
- MPEG-2: 디지털 TV, DVD 등의 고화질 및 고음질을 위한 동영상 압축 규약
- MPEG-3: 고화질 TV 품질에 해당하는 고선명도의 화질을 얻기 위한 기술, MPEG-2와 통합됨
- MPEG-4: 초당 64Kbps, 19.2Kbps의 저속 전송이 가능하도록 압축하는 규약
